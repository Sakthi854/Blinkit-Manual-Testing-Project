# Blinkit Manual Testing Project

## Project Overview

This project demonstrates end-to-end Manual Testing of the Blinkit Mobile Application. The objective was to validate the application's functionality, usability, navigation, and overall user experience across multiple modules by following Software Testing Life Cycle (STLC) practices.

The project includes requirement analysis, test planning, test scenario creation, test case design, test execution, defect reporting, RTM preparation, and project documentation.

---

## Application Under Test

**Application Name:** Blinkit Mobile Application

**Platform:** Android

**Testing Type:** Manual Testing

**Testing Approach:** Functional Testing, UI Testing, Usability Testing, Smoke Testing, Regression Testing

## Testing Activities Performed

### Requirement Analysis

* Explored application features and workflows.
* Identified major modules and functionalities.

### Test Planning

* Prepared Test Plan document.
* Defined scope, objectives, risks, assumptions, and deliverables.

### Test Scenario Design

* Created 108 Test Scenarios covering positive, negative, and usability testing.

### Test Case Design

* Created 108 detailed Test Cases with:

  * Preconditions
  * Test Steps
  * Test Data
  * Expected Results

### Test Execution

* Executed test cases module-wise.
* Recorded Actual Results and execution status.

### Defect Reporting

* Identified and documented defects.
* Assigned Severity and Priority.

### Traceability Management

* Prepared Requirement Traceability Matrix (RTM) to ensure complete requirement coverage.
